### search() 方法源码

```java
// LangChain4jServiceImpl.java 第177-190行
@Override
public List<EmbeddingMatch<TextSegment>> search(EmbeddingModel embeddingModel, String query,
                                                int maxResult, double minScore, Filter filter) {
    // Step 1: 用户问题向量化
    Response<Embedding> response = embeddingModel.embed(query);

    // Step 2: 构建搜索请求
    EmbeddingSearchRequestBuilder searchBuilder = EmbeddingSearchRequest.builder();
    searchBuilder.queryEmbedding(response.content())
            .maxResults(maxResult)
            .minScore(minScore);
    if (filter != null) {
        searchBuilder.filter(filter);
    }

    // Step 3: 执行搜索
    EmbeddingSearchResult<TextSegment> result = embeddingStore.search(searchBuilder.build());
    return result.matches();
}
```

## 二、PgVector 最终执行的 SQL

`PgVectorEmbeddingStore.search()` 内部生成的 SQL：

```sql
SELECT embedding_id,
       text,
       metadata,
       1 - (embedding <=> $1) AS score        -- 计算相似度得分
FROM embedding
WHERE (metadata ->> 'kb_id')::bigint IN (1, 3) -- ① metadata 过滤：只搜指定知识库
  AND 1 - (embedding <=> $1) >= 0.5           -- ② 相似度阈值过滤
ORDER BY embedding <=> $1 ASC                  -- ③ 按距离升序（越近越靠前）
LIMIT 3;                                       -- ④ 最多返回 3 条
```

其中 `$1` 是参数化的查询向量（768 维 float 数组）。

### SQL 各子句解析

#### ① metadata 过滤

```sql
(metadata ->> 'kb_id')::bigint IN (1, 3)
```

- `metadata` 列类型为 JSONB
- `->>` 操作符提取 JSON 字段值为文本
- `::bigint` 转为整数类型
- `IN (1, 3)` 只匹配指定知识库 ID

对应 Java 代码中的 `new IsIn("kb_id", ids)` Filter。

#### ② 余弦距离计算

```sql
embedding <=> $1
```

`<=>` 是 pgvector 提供的**余弦距离**操作符。

#### ③ 相似度得分

```sql
1 - (embedding <=> $1) AS score
```

余弦距离 = 1 - 余弦相似度，所以 `1 - 距离 = 相似度`。

#### ④ 排序 + 截断

```sql
ORDER BY embedding <=> $1 ASC
LIMIT 3
```

按距离从小到大排序（距离越小越相似），取前 3 条。

## 三、余弦相似度的数学原理

### 公式

```
cos_similarity(a, b) = (a · b) / (||a|| × ||b||)

其中:
  a · b   = a[0]×b[0] + a[1]×b[1] + ... + a[767]×b[767]   （点积）
  ||a||   = √(a[0]² + a[1]² + ... + a[767]²)               （L2 范数）

余弦距离 = 1 - cos_similarity
```

### 几何含义

```
                    b
                   /
              θ  /
                /
               O ────────── a

cos(θ) = (a · b) / (||a|| × ||b||)

θ = 0°   → cos = 1.0  → 完全相同方向 → 最相似
θ = 90°  → cos = 0.0  → 正交/无关
θ = 180° → cos = -1.0 → 完全相反方向
```

### 距离与相似度对照表

| 余弦距离 `<=>` | 余弦相似度 | 含义 |
|:---:|:---:|------|
| 0 | 1.0 | 完全相同 |
| 0.1 | 0.9 | 非常相关 |
| 0.3 | 0.7 | 相关 |
| 0.5 | 0.5 | 勉强相关（项目默认阈值） |
| 0.8 | 0.2 | 几乎无关 |
| 1.0 | 0.0 | 完全无关 |

### 为什么用余弦相似度而不是欧氏距离

经过 L2 归一化后，向量都在单位球面上：
- **余弦相似度** = 只看方向，不看长度 → 衡量语义相关性
- **欧氏距离** = 方向 + 长度 → 归一化后等价于余弦距离

PgVector 的 `<=>` 操作符使用余弦距离，与归一化后的 Embedding 向量天然匹配。

## 四、完整检索示例

### 数据状态

假设 PgVector 中已有 3 条向量数据：

| kb_id | text | embedding（简化） |
|:---:|------|------|
| 1 | "Redis连接需要配置host和port" | [0.10, -0.03, 0.08, ...] |
| 1 | "MySQL默认端口3306" | [0.01, 0.15, -0.02, ...] |
| 3 | "Spring Boot整合Redis的配置方法" | [0.09, -0.05, 0.12, ...] |

### 检索过程

**用户提问**："如何配置Redis连接？"

```
Step 1: 向量化用户问题
───────────────────────────────────────
"如何配置Redis连接？" → embed() → 查询向量 Q = [0.11, -0.04, 0.09, ...]

Step 2: 过滤 + 计算距离（WHERE 子句）
───────────────────────────────────────
Filter: IsIn("kb_id", [1, 3])
  → 3条记录全部满足 kb_id ∈ {1, 3}

计算每条与查询向量 Q 的余弦距离:

  Row1: "Redis连接需要配置host和port"
        <=> Q = 0.08  → score = 1 - 0.08 = 0.92  ✓ >= 0.5

  Row2: "MySQL默认端口3306"
        <=> Q = 0.87  → score = 1 - 0.87 = 0.13  ✗ < 0.5  被过滤掉

  Row3: "Spring Boot整合Redis的配置方法"
        <=> Q = 0.15  → score = 1 - 0.15 = 0.85  ✓ >= 0.5

Step 3: 排序 + 截断（ORDER BY + LIMIT）
───────────────────────────────────────
Row1: score=0.92  排第1
Row3: score=0.85  排第2

LIMIT 3 → 返回 2 条（满足条件的只有 2 条）

Step 4: 返回结果
───────────────────────────────────────
[
  EmbeddingMatch { score=0.92, text="Redis连接需要配置host和port" },
  EmbeddingMatch { score=0.85, text="Spring Boot整合Redis的配置方法" }
]
```

### 为什么 "MySQL默认端口3306" 被排除

虽然都是数据库配置文本，但语义上 "MySQL端口" 与 "Redis连接配置" 差距较大。余弦距离 0.87（相似度 0.13）远低于 0.5 的阈值，说明 Embedding 模型正确区分了不同技术主题。

## 五、Filter 的传递路径

### Java 层

```java
// AiChatServiceImpl.java 第114行
new IsIn("kb_id", ids)    // ids = [1, 3]

    ↓ 传入 EmbeddingSearchRequest.builder().filter()

    ↓ PgVectorEmbeddingStore.search() 内部把 Filter 转为 SQL
```

### SQL 层

```sql
WHERE (metadata ->> 'kb_id')::bigint IN (1, 3)
```

`metadata` 列是 JSONB 类型，`->>` 提取 JSON 字段值为文本，`::bigint` 转为整数，再与 `IN` 列表比较。

这实现了**在向量相似度搜索的基础上叠加元数据过滤**，确保只检索用户指定知识库的数据。

## 七、向量索引与性能

### 当前状态

项目未在 PgVector 表上创建向量索引，每次查询都是**精确搜索（暴力扫描）**。

### 数据量增长后的优化方向

```sql
-- IVFFlat 索引（适合 10万+ 数据）
CREATE INDEX ON embedding USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100);

-- HNSW 索引（适合 100万+ 数据，查询更快）
CREATE INDEX ON embedding USING hnsw (embedding vector_cosine_ops);
```

创建索引后，PgVector 的 `<=>` 操作会自动使用索引加速，无需修改 Java 代码。
